The first day of the new blog

In the past, I built my website using WordPress. However, I found its complexity and the multitude of distractions—like figuring out backups and managing SSL certificates for enhanced security—prevented me from focusing on writing. I’m hopeful that this new endeavor will allow me to concentrate more on the content itself.

Let’s dive into the process of setting up this new platform:

Using NVM (Preferred way)

Before using nvm, uninstall the node you have installed(optional).

nvm is a tool that allows you to easily manage different versions of Node. Therefore, on Mac, it is recommended to first install nvm. However, this step is optional if you do not frequently need to switch between different versions of Node.js.

Check this website to install nvm:
https://github.com/nvm-sh/nvm

After installing the nvm, install the latest LTS node version.

This command automatically installs the most recent LTS version of Node.js.

Check the nvm list:

Look for the LTS version you’ve installed, which is marked with “default” or “lts/*” (e.g., “lts/Fermium” for version 14.x.x). Then, set this version as the default for new shells:

This command tells nvm to always use the latest LTS version as the default.

nvm makes handling Node and npm versions easy. When you switch Node versions, nvm also switches npm to the right version. If you upgrade npm, it only changes for that Node version. Removing a Node version with nvm also clears out its npm and packages, keeping things simple and neat.
