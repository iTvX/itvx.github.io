    1047. Remove All Adjacent Duplicates In String

    这道题相当有意思，这道题的解决方案虽然没有直接使用栈，但其核心思想是基于栈的。更进一步，它与双端队列（deque）有相似之处，我们可以用StringBuilder来模拟deque的功能。具体操作如下：

    1. 对于输入字符串S中的每一个字符s，我们都会考虑将其添加到StringBuilder的末尾。
    2. 在添加前，我们首先检查StringBuilder的末尾字符（这可以被看作是deque的头部）是否与当前字符s相同。
    3. 如果末尾字符与s相同，这意味着我们发现了一对重复字符。因此，我们只需删除StringBuilder的末尾字符，模拟了从deque的头部移除的操作。
    4. 如果不同，我们就将字符s添加到StringBuilder的末尾，模拟了向deque的尾部添加的操作。
    5. 通过上述操作，我们实际上是在检查并移除StringBuilder末尾与当前字符s之间的重复字符。

    因此，虽然代码中没有使用到真正的deque数据结构，但其逻辑与操作非常类似，特别是在处理重复字符时，这与从deque的头部添加或移除元素的操作一致。

    459. Repeated Substring Pattern

    https://leetcode.com/problems/repeated-substring-pattern/description/

    这道题，首先要看懂题意，题目的意思是说，一个字符串，可不可以由他本身的子串复制后得到，什么意思呢？比如：abab，可以由ab * 2得到，abc就不行了。就是这个意思。

    为了解决这个问题，我们可以采用一种巧妙的算法。首先假设输入字符串符合题意，即它是由两个或更多相同子串组成的。在这种情况下，如果我们将输入字符串向右移动一位，那么它仍然可以与其后面的字符串对齐。

    具体实现方法如下：

    1. 将输入字符串与自身相加，创建一个包含两个输入字符串的新字符串。
    2. 在新字符串中，从索引 1 处开始查找输入字符串。这是为了避免找到与输入字符串完全相同的前缀。
    3. 如果在新字符串中找到了输入字符串，那么输入字符串一定是由两个或更多相同子串组成的，返回 true。否则，返回 false。

    区分pre/in/post order遍历

    方法：不管tree有多少个节点，永远从宏观上把tree看成只有三个节点：left/root/right。一旦深如到那颗树的里面，从微观再把tree分为三个节点，以此类推。

    也就是说：先将整个树看作一个整体，然后递归地将树分成左、根和右三个子树，直到到达最底层的叶子节点。

    在 Java 中，编译器使用二进制补码( 2’s complement notation)记法来表示有符号整数。所谓二进制补码，就是说最左边的一位表示正数或者负数，0代表正数，1代表负数。对于正数的补码来说，不需要进行任何的操作。对于是负数的补码来说，如果要将一个负数的补码表示法转换回其十进制值，需要按位取反整个补码值（0变为1，1变为0），然后将结果加1，从而得到原始的负数值。

    For example:

    整数-3的二进制补码为11111111111111111111111111111101。为了将其转换回十进制值，需要将其按位取反，得到00000000000000000000000000000010，然后将其加1，得到00000000000000000000000000000011，即十进制数值-3的原始值。
